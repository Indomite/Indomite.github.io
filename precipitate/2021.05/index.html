# 5.6

节后上班，写灵活接页面，在这一次的页面的编写过程中感觉还是会有些进步和收获的，对整体的布局有了思路

实现布局的过程中，先写好类式盒子，写的时候可以想好布局方向，大致想好之后怎么布局

这个地方在移动端的H5布局上，flex弹性布局显得就很重要了，考虑到，当前的元素的排布和边距之间的关系，选择何种布局方式

会形成一定的思路吧，自己在布局的时候也是会有所考虑和联想的

有必要总结一下flex布局

# 5.7

全调度页面完整版，这个项目的话总的来说，还是自己一个人搞定的，在之后跳转页面的时候，选择的还是使用vue-router实现多个页面，在页面的跳转过程中，通过上次的状态控制页面的跳转思路，中间引来的很多问题，（组件之间的状态不好维护，当多个页面组件需要展示的时候，更多的使用if去判断页面是否展示），整体逻辑较为混乱。

在接下来的开发中，还是选择添加路由，这样在设置title和跳转请求数据之间不会有数据未更新等问题

# 5.8

看了楼前端部门的逻辑代码，然后就觉得自己很菜了，整个项目业务复杂，其中的 样式和动画也是很多的，一个页面的逻辑确实需要理很久，不过最后还是借鉴上了之前的一个动画代码，具体思路大致了解了

然后下午的UI走查，其实还是比较重要的一个点吧，感觉自己的样式写的还是不是很极致，在一些属性的使用上还是有所欠缺，比如说一个两个像素之间的差别，这些还是很关键的吧

接下来就是数据的处理了，由于展示的数据样式比较多，所以在一些展示上还是需要下点功夫的，尤其是金额的展示，数字的显示等等，需要做一些数据处理
